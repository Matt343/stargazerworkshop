## [stargazer-workshop] Content model for posts and tracks

Landed on a structure where each post has a general intro (this bit) followed
by track-scoped updates. The updates are parsed out into their own collection
so a project's story can be read in isolation.

The seam is just an H2 with a bracketed track slug, parsed by a custom loader:

```ts
const UPDATE_HEADING = /^##\s+\[([a-z0-9][a-z0-9-]*)\]\s*(.*)$/;

for (const line of lines) {
  const match = line.match(UPDATE_HEADING);
  if (match) flush();
}
```

Feels good so far — one file per post, no separate files to juggle.
